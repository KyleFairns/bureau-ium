Processes the config.json in the parent of the features folder Checks stated profile exists, or uses the framework default. driver

Uses the config to create a browser for the specifications asked for. (this will only work when paired with something like BrowserStack - remote connection to a virtual machine with the browsers specified, handled by the startDriver function, and will run the Selenium code through it natively). Could also add parallel functionality within the framework itself after doing a re-haul, whereby a parallel executor will wrap tests (this may end up being a mocha only feature, but with more thought may work for cucumber)
